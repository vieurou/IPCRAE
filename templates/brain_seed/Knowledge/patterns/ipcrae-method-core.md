# Pattern — Noyau méthodologique IPCRAE

## Modèle mental
- Source de vérité dans les fichiers locaux.
- Séparation brut/digéré (Ressources vs Knowledge/Zettelkasten).
- 3 couches: stockage (vault), IA (bridge), interface (CLI).

## Pipeline
1. capture (Inbox)
2. qualification
3. planification (Projets/Casquettes/Phases)
4. exécution
5. consolidation (memory/Knowledge)
6. audit et auto-amélioration

## Règles critiques
- Traçabilité obligatoire.
- Vérification systématique (tests/commandes).
- Clôture avec mémoire + commit.
